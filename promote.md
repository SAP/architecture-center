## Terraform - Infrastructure as Code

Chances are that you already rely on Terraform to manage your cloud
infrastructure. You have described your infrastructure in declarative code and
put it under version control with Git to keep a history log of the
infrastructure state. Because every change to the infrastructure, which is done
via Terraform, gets documented, it is possible to rollback to a previous state.

If this sounds familiar to you, and you decided that you want to build on the
shown reference architecture, please checkout the Terraform files linked below.
They include Terraform resources for most of the infrastructure from the
reference architecture.

If, on the other hand, you are new to Terraform but want to learn more, click
[here](https://sap-docs.github.io/terraform-landingpage-for-btp/) to get
started with Terraform on SAP BTP.
