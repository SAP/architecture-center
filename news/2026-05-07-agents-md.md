Our community has been leveraging AI (coding) agents for a good while now to contribute to SAP Architecture Center. From changing the look and feel of the site for a [fresh experience](./2026-04-22-introducing-the-refreshed-sap-architecture-center.md) to putting final touches on reference architectures. So it was a natural next step to try grounding these AI agents with basic context about SAP Architecture Center itself for better, immediate answers aligned with the project. In fact, the corresponding repository on GitHub has a new [AGENTS.md](https://github.com/SAP/architecture-center/blob/dev/AGENTS.md) as of recently, which does just that.

## Motivation for AGENTS.md

`AGENTS.md` is essentially a context file for AI agents described in simple Markdown, and a new standard developed under the [Agentic AI Foundation](https://aaif.io). The file may have a project description; specify the used (development) framework; list common commands and applied conventions for file names and code style etc, **but it certainly doesn't have to**. In the end it's about adding what helps the agent produce the desired results for the project.

<!-- ![excerpt from our AGENTS.md](img/2026-05-11/excerpt-agents-md.png) -->

Many AI agents such as opencode, OpenAI Codex, and GitHub Copilot have adopted the standard. They look for an `AGENTS.md` file and make sure to load its instructions into the starting context of the agent for every new session. So there's never been a better time to clone our repository and start exploring it (perhaps even contribute to it) through your AI agent of choice.

![core idea of AGENTS.md](img/2026-05-11/agents-md-core-idea.png)

Also, don't forget to check out the new page on [AI agents](/docs/community/get-started-ai-agents) as part of our community of practice, which has some examples to ask the agent now that there's an `AGENTS.md`. With that, we hope you found this interesting and until next time!
